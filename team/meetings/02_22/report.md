# Réunion 2 | 22 février
## Décisions prises
### Pair programming
- Pour le premier binôme on choisit quelqu'un qu'on connait pour s'habituer/voir comment le pair programming fonctionne
- On change de binôme toutes les semaines
- Pas de rôle dédié au support, s'il y a un problème, il y a <#1468667078075748493> et <#1468666591100145835> pour poser des questions et ceux qui peuvent aident
### Conventions de code
- On suit les conventions déjà faites comme dans le doc envoyé par Daniel Cabeça (qui va aussi faire un document pour résumer les conventions qui nous sont utiles)
- On utilise les tab et pas les espaces
- Pour javadoc, on complète @param, @return et @throws
- Pas de tests pour les constructeurs, getters et setters, sauf si le constructeur est plus "complexe" ou renvoie des exceptions par exemple
- Pas d'IDE obligatoire, chacun utilise ce qu'il veut
### Structure d’organisation du code source
- Comme on doit suivre le MVC, on peut structurer le code avec des dossiers/packages model, view et controller (et utils)
- On peut adapter selon les besoins quand on va commencer à coder
### Format des commits
- Le co-auteur doit être précisé selon un format spécifique
- On utilise une structure avec des keywords (add, edit, remove)
### User stories
- Pour attribuer les stories, c'est premier arrivé, premier servi
- Quand on commence à travailler sur une story, on doit le préciser dans <#1473723613520593080>
- Si après une session, la story/tâche n'est pas terminée, quelqu'un d'autre peut la reprendre
